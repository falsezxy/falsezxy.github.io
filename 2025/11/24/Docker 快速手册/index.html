第二部分：Docker 常见面试题

1. 基础概念

Q1：什么是 Docker？Docker 的优势是什么？

为什么要用docker

1.容器相对于虚拟化或者传统模式而言，运行起来更方便

2.启动速度快，更适合档当前企业的运行模式——需要快速更新，快速上线，方便管理

答案： Docker 是一种容器化技术，允许开发者将应用及其依赖打包到一个标准化的容器中。

优势：

  • 轻量级：容器共享宿主机内核，不需要完整的操作系统
  • 快速启动：秒级启动，远快于虚拟机
  • 隔离性：进程、网络、文件系统隔离
  • 一致性：解决”在我的机器上可以运行”的问题
  • 易于扩展：快速部署和扩展
  • 资源利用率高：相比虚拟机效率更高

Q2：Docker 与虚拟机的区别

答案：

特性 Docker 容器 虚拟机
启动速度 秒级 分钟级
CPU 损耗 几乎无损耗 ~50% 损耗
性能 接近原生 弱于原生
系统支持量（单机） 上千个 几十个
隔离性 资源隔离/限制 完全隔离
架构 操作系统级虚拟化 硬件级虚拟化

关键差异： 虚拟机每个都包含完整操作系统，而 Docker 容器共享宿主机内核。

Q3：Docker 的核心技术是什么？

答案：

  • Namespace（命名空间）：实现资源隔离，6 个命名空间
    • PID：进程隔离
    • NET：网络隔离
    • MNT：文件系统挂载隔离
    • IPC：进程间通信隔离
    • UTS：主机名隔离
    • USER：用户名隔离
  • Cgroup（控制组）：实现资源限制
    • CPU 限制
    • 内存限制
    • I/O 限制
    • 设备访问控制
  • UnionFS（联合文件系统）：实现分层镜像
    • AUFS、OverlayFS、Devicemapper

Q4：Docker 的三个核心组件是什么？

docker镜像——特殊的模板文件，类比centos操作系统镜像

docker容器运行时——用于描述持续运行的容器

docker镜像仓库——类似yum仓库

答案：

  1. Docker 镜像（Image）
    • 只读模板，包含应用及依赖
    • 由多层文件系统组成
  2. Docker 容器（Container）
    • 镜像的运行实例
    • 隔离的运行环境
    • 在镜像基础上添加可读写层
  3. Docker 仓库（Registry）
    • 存储和分发镜像的中心
    • Docker Hub（公共）
    • 私有仓库（Private Registry）

2. Docker 镜像相关

Q5：镜像如何分层？为什么要分层？

答案： 分层原理：

  • 每一条 Dockerfile 指令都创建一个新的镜像层
  • 镜像由多个只读层叠加而成
  • 容器启动时，在镜像顶部添加可读写层

分层优势：

  • 提高复用性：相同的层可被多个镜像共享
  • 提高构建速度：Docker 缓存每一层，修改时只需重建该层及以后的层
  • 节省空间：相同层只存储一份
  • 便于维护：每层职责清晰

Q7：为什么 Docker 中的 CentOS 只有 200MB，而完整的 CentOS 需要几 GB？

答案：

  • 精简的 rootfs：Docker 镜像只包含基本命令、工具和程序库，不包含完整的系统文件
  • 共享内核：容器直接使用宿主机的内核，无需在镜像中包含内核
  • 分层存储：多个镜像可共享相同的基础层，节省空间
  • bootfs 一致性：不同 Linux 发行版的 bootfs 基本一致，rootfs 会有差异

Q8：如何优化 Dockerfile 构建？

答案：

  1. 减少层数

  2. 利用缓存

    • 将频繁改变的指令放在后面

  3. 使用 .dockerignore

    • 排除不必要的文件

  4. 使用多阶段构建

3. 容器相关

Q9：容器的生命周期和主要状态有哪些？

答案： 7 个主要状态：

  1. created：已创建，未启动
  2. running：运行中
  3. paused：暂停状态
  4. restarting：重启中
  5. removing：迁移中
  6. exited：停止状态
  7. dead：死亡状态（系统异常或断电）

关键点： 容器依赖 PID=1 的进程，若该进程退出，容器自动停止。

Q10：Docker 容器保持运行的关键是什么？

答案： 容器必须有一个前台进程（PID=1）持续运行。当该进程退出时，容器也随之停止。

Q11：pause 和 stop 的区别

答案：

  • docker pause：暂停容器中所有进程，容器状态为 paused，资源占用不释放
  • docker stop：向容器内主进程发送 SIGTERM 信号（默认 10s 超时），然后 SIGKILL，容器完全停止，可重启

4. 网络相关

Q13：Docker 网络的四种模式是什么？

答案：

  1. bridge（桥接）
    • 默认模式
    • 容器通过虚拟网桥 docker0 通信
    • 容器拥有独立 IP 地址
    • 与宿主机网络隔离
  2. host（主机）
    • 容器与宿主机共享网络栈
    • 无独立 IP，使用宿主机 IP 和端口
    • 性能最好但隔离性最弱
  3. container（容器）
    • 两个容器共享同一网络栈
    • 共享 IP 地址和端口范围
    • 其他资源仍隔离
  4. none（无网络）
    • 容器无网络配置
    • 仅有 lo 回环接口
    • 最安全但无法联网

Q14：端口映射中 -p 和 -P 的区别

答案：

  • -p：指定映射端口

  • -P：随机映射端口

Q15：Docker 跨主机通信如何实现？

答案：

  1. Overlay 网络（Docker Swarm）
    • 跨主机容器通信
    • 创建覆盖网络
  2. Macvlan
    • 容器获得独立 MAC 地址
    • 直接连接物理网络
  3. Flannel（Kubernetes）
    • 为 Pod 分配跨节点 IP

Q16：Docker 容器间如何通信？

答案：

  1. 同宿主机
    • bridge 模式：通过 docker0 网桥通信
    • container 模式：共享网络栈
    • –link 选项：通过容器名访问
  2. 跨宿主机
    • Overlay 网络
    • Macvlan 模式
    • 容器编排平台（Swarm/Kubernetes）

5. 数据管理

Q17：Docker 数据持久化的三种方式

答案：

  1. 数据卷（Volumes）

    • 推荐方式
    • Docker 管理存储位置
    • 性能最好

  2. 挂载宿主机目录（Bind mounts）

    • 挂载宿主机指定目录
    • 方便备份和共享

  3. tmpfs（临时文件系统）

    • 存储在内存中
    • 容器停止后数据丢失
    • 适合临时数据

Q20：cgroup 有什么作用？

答案： cgroup（Control Groups）是 Linux 内核提供的资源控制机制，Docker 通过它实现资源限制。

四大功能：

  1. 资源限制：限制任务使用的总资源量（CPU、内存、I/O）
  2. 优先级分配：分配 CPU 时间片和 I/O 带宽优先级
  3. 资源统计：统计 CPU 时长、内存用量等
  4. 任务控制：对 cgroup 中的进程执行暂停/恢复/终止

7. Docker Compose

Q21：Docker Compose 是什么？为什么需要它？

答案： Docker Compose 是 Docker 官方的容器编排工具，用于定义和运行多个容器应用。

为什么需要：

  • 简化多容器应用的启动和管理
  • 定义容器间的依赖关系
  • 实现服务的一键部署
  • 支持环境变量和覆盖
