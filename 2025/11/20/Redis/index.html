Redis 服务控制
/etc/init.d/redis_6379 stop #停止
/etc/init.d/redis_6379 start #启动
/etc/init.d/redis_6379 restart #重启
/etc/init.d/redis_6379 status #状态

修改配置 /etc/redis/6379.conf 参数，并重启服务，完成redis部署

  • bind 127.0.0.1 → 指定监听 IP
  • port 6379 → 默认端口
  • daemonize yes → 守护进程模式
  • logfile /var/log/redis.log → 日志路径
  • requirepass 123456 → 密码 不用管

举例：
Redis 就像一个仓库 → bind 是仓库大门只允许谁进，port 是大门编号，daemonize 表示仓库是否一直有人值守，requirepass 是仓库的锁。

工作流程

redis 主从复制
1、从向主发送sync请求
2、主开启bgsave进行RDB 持久化（保存文件）
3、保存后推送给从服务器，从服务器开始”恢复”数据到内存中
4、同时主将内存中新增的数据通过AOF的方式给到从服务器

RDB相当于快照，下次启动时恢复

AOF记录执行语句，启动时重新执行，以文件形式存储
