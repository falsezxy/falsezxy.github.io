Docker 构建前后端项目 RuoYi

一、项目概述

什么是 RuoYi？

RuoYi 是一个开源的前后端分离式管理系统，提供了完整的企业级应用框架。它包含：

  • 前端：Vue.js 构建的单页面应用（SPA）
  • 后端：Spring Boot 的 RESTful API 服务
  • 数据库：MySQL 数据存储
  • 缓存：Redis 缓存系统

应用场景

场景 使用方式
OA系统 在 RuoYi 基础上添加请假、审批、工资条管理
ERP系统 添加采购、销售、库存、财务等模块
CRM系统 添加客户、销售机会、合同管理
电商后台 添加商品、订单、库存、营销管理
学校管理系统 添加学生、成绩、课程管理

为什么选择 RuoYi？

  • 开箱即用：不用从零开始写权限、菜单、日志等基础功能
  • 完整文档：有详细的使用和二次开发文档
  • 活跃社区：遇到问题有人帮忙解答
  • 代码质量高：遵循企业级开发规范
  • 前后端分离：便于团队协作，前端和后端独立开发

二、系统架构与数据流

关键通信流程

  1. 用户请求 → 浏览器访问 192.168.110.128:8888
  2. Nginx 处理 → 如果是前端资源，直接返回；如果是 API 请求（/prod-api/），转发给后端
  3. 后端处理 → Spring Boot 应用处理业务逻辑，访问 MySQL 和 Redis
  4. 返回结果 → 前端渲染，展示给用户

三、技术栈与运维重点

后端核心框架：Spring Boot

  • 优势：内置 Tomcat，打包成单个 JAR 文件，无需额外配置 Web 服务器
  • 运维要点：需要在目标服务器上安装正确版本的 Java Runtime Environment（JRE）

安全框架：Spring Security

  • 功能：提供开箱即用的认证和授权
  • 运维要点：
    • 理解会话管理机制（默认使用 Session）
    • 配置 JWT 令牌用于集群部署时的会话共享

数据访问层：MyBatis

  • 运维关注：维护 MySQL 数据库
  • 具体工作：
    • 数据库安装、用户权限管理
    • 定期备份与恢复策略
    • SQL 慢查询监控与优化

缓存系统：Redis

  • 关键用途：存储会话、验证码和系统缓存
  • 高可用配置：
    • 主从复制：保证数据备份
    • 哨兵（Sentinel）或集群（Cluster）：确保服务高可用
  • 数据持久化：
    • 配置 RDB 或 AOF 策略，防止数据丢失
  • 监控指标：内存使用率、连接数、命中率

工作流程

工作流程：

  1. 用户通过浏览器访问Nginx服务
  2. Nginx返回静态前端资源（HTML、CSS、JS）
  3. 前端发起API请求被Nginx代理到后端
  4. 后端处理业务逻辑，访问MySQL数据库和Redis缓存
  5. 返回结果给前端并渲染展示

四、部署与实践

部署流程简述

1. 环境准备

  • 安装Docker和docker-compose
  • 创建工作目录 /opt/ruoyi
  • 拉取所需镜像（nginx、mysql、redis、node、maven）

2. 前端构建

  • 用Node容器执行 npm install 和 npm run build:prod
  • 生成dist目录（静态文件供Nginx使用）

3. 后端打包

  • 用Maven容器执行 mvn package
  • 生成ruoyi-admin.jar文件

4. 构建Java镜像

  • 编写Dockerfile
  • 基于CentOS 7 + JDK 1.8
  • 将jar包复制到镜像中

5. 配置Nginx

  • 编写nginx.conf配置文件
  • 配置反向代理到后端Java服务（java.com:8080）

6. 编写docker-compose.yml

  • 定义4个服务：ruoyi-web、java、db、redis
  • 配置容器间的links和depends_on
  • 挂载配置文件和数据卷

7. 启动

  • 执行 docker-compose up
  • 访问 http://IP:8888 使用系统

五、常见问题排查

问题：后端无法连接到数据库

原因：

  • 后端配置文件中数据库密码（password）与 docker-compose.yml (123456)中设置不一致
  • 配置文件中的地址不是容器名称

解决：

  1. 检查 application-druid.yml 中的密码设置
  2. 确保密码与 docker-compose.yml 中 MYSQL_ROOT_PASSWORD 一致
  3. 数据库地址应该使用容器名称（如 mysql.server）而不是 IP

问题：前端 API 请求失败

原因：Nginx 代理配置错误或后端服务未启动

解决：

  1. 检查 nginx.conf 中的 proxy_pass 地址是否正确
  2. 确认后端容器正常运行：docker ps 查看容器状态
  3. 查看 Nginx 日志：docker logs ruoyi-web

六、监控与维护

应用监控

  • RuoYi 内置 Druid 监控，可查看 SQL 执行情况
  • 集成 Spring Boot Actuator + Prometheus + Grafana 可实现 JVM 性能和应用健康度的可视化监控

系统监控

  • 使用 Zabbix 或 Prometheus 监控服务器 CPU、内存、磁盘 I/O 和网络流量

日志收集

  • 配置 Logback 或 Log4j2 将应用日志输出到文件
  • 使用 ELK（Elasticsearch, Logstash, Kibana）或 EFK 栈进行日志收集、存储和检索

高可用配置

  • 应用层：部署多个无状态 Spring Boot 实例，通过 Nginx 负载均衡
  • 数据库：MySQL 配置主从复制或 MHA 高可用方案
  • 缓存：Redis 配置哨兵或集群模式

备份策略

  • 定期执行 MySQL 全量与增量备份
  • Redis RDB 快照备份
  • 应用服务器文件备份
