Harbor 私有镜像仓库完整部署指南

简介

Harbor 是由 VMware 开源的企业级容器镜像仓库解决方案，支持镜像存储、版本管理、访问控制等丰富功能。本文将详细介绍如何搭建一套完整的 Harbor 私有镜像系统。

第一步：环境准备

准备好Docker

第二步：核心配置

修改 Harbor 配置文件

必需参数配置

参数名 说明 示例
hostname Harbor 访问地址，支持 IP 或域名 192.168.118.101 或 hub.example.com
ui_url_protocol UI 访问协议 http 或 https
harbor_admin_password 管理员初始密码 Harbor12345
db_password MySQL 数据库密码 自定义安全密码
max_job_workers 镜像复制作业线程数 10

可选参数详解

认证相关： 支持数据库认证（db_auth）和 LDAP 认证（ldap_auth），默认使用数据库存储凭证。

用户管理： 可通过 self_registration 参数控制是否允许用户自注册，禁用后仅管理员可创建新用户。

权限控制： 使用 project_creation_restriction 参数限制项目创建权限，设为 “adminonly” 时只有管理员可创建项目。

令牌有效期： Token_expiration 参数控制由令牌服务生成的令牌过期时间，默认 30 分钟。

远程证书验证： verify_remote_cert 参数决定是否验证远程仓库的 SSL/TLS 证书，自签名证书可设为 off 跳过验证。

存储后端配置

默认情况下，Harbor 将镜像数据保存在本地文件系统 /data/registry 目录。生产环境建议配置对象存储服务（如 S3、Ceph）以提升可扩展性，需修改 common/templates/registry/config.yml 文件。
