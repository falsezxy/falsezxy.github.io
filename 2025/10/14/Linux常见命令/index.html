Linux的目录结构与常见命令

一、引言

了解Linux命令前，首先要知道输入命令的工具shell是什么，命令有哪些命令?

Shell

shell是Linux系统中后台运行的一种特殊程序也可以理解成一种特殊的软件，提供了用户与内核进行交互操作的一种接口。通俗讲，用户使用应用程序发布命令（比如使用浏览器访问网页），Shell去通知操作系统需要进行什么操作（相当于解释层）。

命令种类

shell 分为内部命令和外部命令

内部命令：在系统启动时就调入内存中，是常驻内存的，执行效率非常高。

外部命令：是系统软件的功能，用户需要时才从硬盘读取到内存中。

一般先执行内部命令再执行外部命令（别名最大）。

二、目录结构

了解完使用命令的工具，其次就需要了解Linux系统中，什么目录是做什么的，存放什么内容。

Linux的目录结构如下图：

1) / (根目录) ⭐

整个文件系统的起点

2) /bin ⭐

存放基本的系统命令，如ls, cp, mv

3) /boot

包含引导加载程序文件和内核

4) /dev ⭐

设备文件目录，如硬盘、键盘等

5) /etc ⭐

系统配置文件目录

6) /home

用户主目录

7) /mnt

临时文件系统挂载点，/data1、 /data2 、 /backup

8) /opt

可选应用软件包

9) /root ⭐

root用户的主目录

10) /sbin

系统管理命令，如fdisk, ifconfig

11) /tmp

临时文件目录

12) /usr ——主要存放一些自定义程序

用户程序和数据 /usr/local存放一些安装程序（免安装的应用，绿色软件，源代码自定义安装）

13) /var ⭐

可变数据，如日志文件/var/log

messages：Linux的日志垃圾桶

secure：存放安全有关的日志

18) /run

运行时变量数据

~：当前所在的位置

#：管理员权限 – $：普通用户权限

3.ls

ls:查看目录内容 语法:ls [选项] 目录

常用选项:

​ -l #将文件信息属性详细列出，通常直接使用ll，因为设置了别名alias(别名使用自行了解)

​ -R #递归显示，比如一个目录下还有文件，则把那个文件也一起列出

​ -d #查看目录本身信息，不显示目录下的内容

​ -h #友好显示文件大小（human-readable）

​ -a/A #遍历所有文件及目录 (ls内定将文件名或目录名称开头为”.”的视为隐藏档，不会列出)

了解详细信息前各个字母的含义

第一个字符：代表文件类型 -普通文件 d目录

后续每三个字符为一组，分别代表：所有者权限，用户组权限，其他用户权限

r代表读取权限：所有者可以打开和读取文件。
w代表写权限：所有者可以修改文件。
x代表执行权限：所有者可以运行文件（如果它是程序或脚本）。

通配符扩充
符号 效果
？ 匹配一个字符 f?.txt
* 匹配所有非隐藏的字符无论多长多短，但不匹配 “.” 开头的文件，即隐藏文件
{1..10} 1到10
{a..z} a b c …z
{A..Z} A B C …..Z
[123] 取其中之一有就显示没有也无所谓
[a-z] 范围需要注意
[0-9] 匹配数字范围(0123456789)
\ 转义符，表示原来的意思
[[:lower:]] 小写字母表示 a-z
[[:upper:]] 大写字母表示 A-Z
[^zhu] 匹配列表中的所有字符以外的字符
[[:diogit:]] 任意数字，相当于0-9

15.find

find:在指定目录下查找文件和目录,可以使用不同的选项来过滤和限制查找的结果 语法:find 查找范围 [关键字/选项] 查找内容

常用选项:

​ -name:按名称查找，支持使用通配符 * 和 ?。

​ -size: 按大小查找，支持使用 + 或 - 表示大于或小于指定大小，单位可以是 c（字节）、w（字数）、b（块数）、k（KB）、M（MB）或 G（GB）。

​ -user:按文件所有者查找

​ -type:按类型查找：普通文件（f）、目录（d）、块设备文件（b）、字符设备文件（c）、链接（l）

​ -mtime days：按修改时间查找，支持使用 + 或 - 表示在指定天数前或内，days 是一个整数表示天数。

​ -ctime days: 查找在 n*24 小时内状态发生变化的文件（例如权限）。

​ -atime days: 查找在 n*24 小时内被访问过的文件。

​ -exec {} \ : find 命令的执行选项，-exec(执行选项) 要执行的命令(比如ls rm等) {}(占位符，找到的文件) \(结束符，代表结束，必须要有)

​ -delete : 在find命令的最末尾使用，删除查找的所有文件

块设备指的是成块读取数据的设备（如硬盘、内存等），而字符设备指的是按单个字符读取数据的设备（如键盘、鼠标等）

拓展：find命令选项可以结合逻辑运算符使用

-a表示而且and

-o表示或者or

什么是归档？

文件归档就是将一组拥有相同属性的文件或目录组合到一个文件中，并从新存储它们到一个专门的目录中，以便以后在需要时进行查阅。

​ -z:此选项告诉tar使用gzip压缩文件/正在处理一个gzip文件

​ -j:此选项告诉tar使用bzip2压缩文件

​ -t：列出(list)归档文件中的内容

常见用法：

1.一步压缩：

①gzip格式压缩

②bzip2格式压缩

2.一步解压: 指定解压后路径 tar -xzvf 归档文件 -C 指定解压到目录

①gzip格式解压

②bzip2格式

3.查看压缩文件中的内容

不解压仅查看 ①-tzvf ②-tjvf

xargs可以将管道或标准输入（stdin）数据转换成命令行参数，也能够从文件的输出中读取数据。

rpm命令的使用方法：

② 删除

24.vi编辑器:文本编辑器

vi的三种模式⭐

①命令模式（Command Mode）

②输入模式（Insert Mode）

③命令行模式（末行命令模式）（Command-Line Mode）

操作方法：
① vi 文件名，进入到vi的文本编辑器中
② 默认位置：命令模式，该模式一般只用于直接阅读或换行查阅
③ 按 “i” ，可以从命令模式，进入到输入模式（insert模式），该模式下，可以进行编辑文本
④ 文本编辑完成后，下一步我们需要保存并退出时，需要先从当前的输入模式按esc切换到命令模式，然后先按住shift 再按 : 进入到末行模式,然后可以输入wq（w：保存、q时退出），完成文件的保存并退出动作

PS：如果不小心删除了xx行或者原文内容，按 u 撤回
如果只想退出，不想保存的话，按esc从输入模式切换到命令模式，然后先按住shift 再按 : 进入到末行模式，最后输入q！

常用操作：

在末行模式：

  • 输入set nu 可以显示行号，输入set nu! 可以取消行号
  • 输入行数，例如1140 ，可以直接跳转到该行
  • 按住shift + G 可以跳转到最后一行
  • 输入 ”/关键词“ 例如 /nginx ，可以跳转到文件中包含该关键词的指定行，按 “n” 可以跳转到包含该关键词的下一行， 按”N”，跳转到上一行

在命令模式：

  • 输入”o” ，可以在光标所在的下一行插入空行，且可以直接编辑
  • 输入”O”，可以在光标所在上一行插入空行，且可以直接编辑
  • 输入”u”，可以直接撤回之前操作行的内容
  • 输入”dd”，可以删除光标所在行的所有内容，可以输入”数字dd” 表示删除从光标所在行开始的第n行，例如删除5行 “5dd”
  • 删除光标所在位置往后的内容：”d$”
  • 在光标所在行的位置，输入”yy” 或者 “数字yy” 可以复制单行/连续复制从光标开始的多行内容，然后可以在想要粘贴的位置，输入p，来将之前复制的内容，粘贴到光标所在行的下行

25.mount

mount:用于挂载Linux系统外的文件 语法:mount 挂载设备 挂载点

什么是挂载？

挂载是Linux系统中的一个核心概念，它是将一个存储设备（如硬盘分区、USB设备、光驱、网络存储等）**关联（共享）**到文件系统目录树中的某个目录上的过程。挂载后，用户就可以通过该目录访问该存储设备中的文件和数据。

挂载块设备到/mnt目录

卸载挂载的设备umount

mount命令是Linux系统管理的基础操作，理解挂载的概念和使用方法对于系统管理员和Linux使用者都很重要。通过合理使用mount命令，可以灵活管理系统中的各种存储设备。

用户常用命令

三种用户

①超级用户：root UID 0

②普通用户：普通登录用户 UID 1000-∞，创建用户时不指定uid，uid自增长

③系统用户/程序用户：通常运行在后台，分别管理各自的系统服务或者应用程序 UID 1~999

用户组

①基本组：用户创建时，会自动创建基本组，默认情况下基本组和用户名是一致的

②附加组：权限的集合，一个用户可以属于多个附加组

用户账号文件/etc/passwd ： 账号信息记录的文件

进程常用命令

操作系统中进程-服务-系统之间的关系

进程：程序的运行实例，最小的资源分配单位。

服务：在后台持续运行的程序，通常由一个或多个进程组成。

系统：操作系统整体，由内核、服务、进程等组成。

进程是实现：服务通过进程来实现功能

服务是管理：系统通过服务来组织和管理进程

系统是整体：系统由多个服务和进程组成

简单说明进程

进程分为：主进程，子进程，线程

**主进程：**主进程是程序运行时创建的第一个进程。负责整个程序的生命周期管理。主进程退出时，程序通常随之结束。拥有独立的内存空间和系统资源。

**子进程：**由主进程创建的新进程。拥有独立的内存空间，与主进程和其他子进程相互隔离。子进程之间互不影响，一个崩溃不会导致其他进程崩溃。

**线程：**进程内部的执行单位。同一进程中的多个线程共享内存空间、文件描述符等资源。线程之间通信简便，数据共享方便但需要同步控制。

简而言之：

  • 进程是独立的“工厂”，资源和任务完全独立。
  • 线程是“工厂”里的“工人”，多个线程共享同一个进程的资源。

常见进程状态Stat：

常规状态:
R 正在运行
S 休眠（可被唤醒）
T 停止状态
异常状态：
僵尸进程：子进程已经结束，但父进程还没有读取它的退出状态，该子进程就称为僵尸进程。
孤儿进程：父进程退出，子进程还在（占用资源） 可手动kill
出现的原因大都是因为系统资源压力过大导致的，常规情况下，系统是可以自行处理以上的异常状态。

一般进程存放在/proc目录中

44.top/htop

① ⭐top：实时系统监控工具，显示系统的整体性能信息以及正在运行的进程的相关信息。

在屏幕的顶部，top 显示了系统的总体状态，包括：

  • uptime：系统启动的时间。
  • load average：系统的负载平均值，通常是过去 1 分钟、5 分钟和 15 分钟的负载。
  • tasks：系统当前的进程数（包括运行中、睡眠中、停止等状态）。
  • CPU：显示 CPU 的使用情况，包括用户态、系统态、空闲等。
  • Mem：显示内存的总量、已用、剩余等。
  • Swap：显示交换空间（swap）的使用情况。

在 top 的底部，列出了系统中所有进程的资源使用情况，包括：

  • PID：进程 ID。
  • USER：进程所属的用户。
  • PR 和 NI：进程的优先级和 nice 值。
  • VIRT：进程使用的虚拟内存量。
  • RES：进程使用的常驻内存量（物理内存）。
  • SHR：共享内存量。
  • S：进程的状态（S = 睡眠，R = 运行，Z = 僵尸进程等）。
  • %CPU：进程使用的 CPU 百分比。
  • %MEM：进程使用的内存百分比。
  • TIME+：进程使用的总 CPU 时间。
  • COMMAND：进程名称或命令。

按 1 可以查看cpu相关信息

参数说明：

  • -d <秒数>：指定 top 命令的刷新时间间隔，单位为秒。
  • -n <次数>：指定 top 命令运行的次数后自动退出。
  • -p <进程ID>：仅显示指定进程ID的信息。
  • -u <用户名>：仅显示指定用户名的进程信息。
  • -H：在进程信息中显示线程详细信息。
  • -i：不显示闲置（idle）或无用的进程。
  • -b：以批处理（batch）模式运行，直接将结果输出到文件。
  • -c：显示完整的命令行而不截断。
  • -S：累计显示进程的 CPU 使用时间。

常用操作：

  • P：按 CPU 使用率排序。
  • M：按内存使用率排序。
  • T：按运行时间排序。
  • k：杀死进程，输入进程 ID 后，选择是否终止该进程。
  • q：退出 top。

其中需要了解⭐load average

Load Average（平均负载）是系统在过去一段时间内，处于运行队列中的进程数的平均值。

load average所显示的三个数字是cpu过去1min 5min 15min的平均使用率

载值范围(设4核心) 系统状态 说明
Load < 核心数(<4) 正常 系统有空闲CPU资源，进程无需等待
Load = 核心数(=4) 满负载 CPU刚好满载，所有核心都在工作，无浪费也无等待
Load > 核心数(>4,=8(两倍)) 高负载 有进程在等待CPU资源，系统响应会变慢
Load 远大于核心数(3倍及以上) 严重过载(随时崩溃) 大量进程等待，系统性能严重下降，需要立即处理
Load 接近 0 空闲 系统资源充足，进程很少

通过这三个数字的趋势，可以判断系统状态，从而做出对应操作。
