磁盘相关知识

1 磁盘分区

1.1 首先需要了解，什么是磁盘分区？为什么要进行磁盘分区？

1、磁盘存储空间无法直接使用，需要先进行分区（划分存储空间）才能使用磁盘容量。分区的主要目的是将一块完整的磁盘逻辑地分割成多个独立的存储区域。
2、主分区只能创建4个（/dev/sda1 /dev/sda2 /dev/sda3 /dev/sda4）磁盘分区创建好后，需要格式化就能使用其空间了。

主分区，主要存储 ： ① 系统文件 ② 程序文件 ③ 用户数据

3、剩余的空间可以自定义控制，但是得先化分扩展分区
比如，一块磁盘一共有1000G，主分区用了400G，还剩余600，此时我想使用剩余的600G空间，需要先把600G的空间定义为扩展分区（扩展分区不能直接存储数据），需要把扩展分区中的存储空间，进一步创建为1个或多个逻辑空间之后，才能作为存储设备使用。

1.2 主分区（Primary Partition）

  • 主要存储系统文件、程序文件、用户数据
  • Linux系统最多只能创建4个主分区
  • 命名规则：/dev/sda1、/dev/sda2、/dev/sda3、/dev/sda4
  • 需要格式化后才能挂载使用

1.3 扩展分区（Extended Partition）

  • 用于分割主分区之外的剩余磁盘空间
  • 无法直接存储数据，仅用于分割存储空间
  • 一块磁盘最多只能有1个扩展分区
  • 扩展分区中的空间需进一步划分为逻辑分区

1.4 逻辑分区（Logical Partition）

  • 从扩展分区分割出来的实际存储分区
  • 可用于用户自定义用途
  • 编号从5开始：/dev/sda5、/dev/sda6等
  • 需要格式化后才能挂载使用

1.5 ⭐交换分区（SWAP）

  • 临时作为内存的存储空间使用
  • 当物理内存不足时，系统会使用SWAP分区
  • 一般只用于内存数据的读取
  • 建议大小为物理内存的1.5～2倍
  • 对于物理内存充足（8GB以上）的服务器可不设置

简单来说，SWAP分区临时作为内存的存储空间使用，一般只用于内存数据的读取

Linux设备文件命名规则

Linux将硬盘、分区等设备均表示为文件，设备文件位于/dev/目录下。

磁盘接口模式 SCSI IDE SATA ——》表现为文件时的命名规则有区别

命名规则

以/dev/sdb5为例：

  • sd：表示SCSI设备（hd表示IDE设备）
  • b：硬盘的顺序号，以字母a、b、c…表示
  • 5：分区的顺序号，以数字1、2、3…表示
    • 主分区编号：1-4
    • 第一个逻辑分区始终从5开始

2.文件系统的概念

文件系统的基本功能是管理文件的生命周期。不同类型的文件系统在应用特性上会有区别。

常用的文件系统：XFS、EXT4、SWAP

2.1 XFS文件系统

  • 高性能日志文件系统
  • 特别擅长处理大文件
  • 支持上百万TB的存储空间
  • 开启日志功能，宕机时数据不易损坏
  • 可根据日志记录快速恢复数据
  • CentOS 7默认使用XFS

3.2 EXT4文件系统

  • 第4代扩展文件系统
  • 用于存放文件和目录数据的分区
  • 典型的日志型文件系统
  • 保存磁盘存取记录的日志数据，便于恢复
  • 存取性能和稳定性出色
  • CentOS 6及更早版本的默认文件系统

3.3 SWAP文件系统

  • 交换文件系统
  • 建立交换分区供Linux系统使用
  • 作用相当于虚拟内存
  • 在一定程度上缓解物理内存不足的问题
  • 不用于直接存储用户文件和目录数据

3.4 其他文件系统

  • JFS：多用于商业版本UNIX系统，性能表现出色
  • FAT16/FAT32：较老的文件系统
  • NTFS：微软公司的专有文件系统，Linux默认仅支持读取，需结合NTFS-3G等辅助软件才能写入

3.磁盘分区操作

磁盘想要给目录使用（共享存储空间）
通常需要经过：① 分区 –》② 格式化 –》③ 挂载
① 分区命令： fdisk 磁盘设备（例如：/dev/sdb）
② 格式化命令：mkfs -t xfs/ext4 设备文件（例如分区）
③ 挂载命令： mount 设备文件 目录位置 （命令的挂载通常，都是临时性的）
永久挂载：写入到配置文件中 /etc/fstab
方式: ① 直接使用设备文件名字进行挂载
② 使用设备UUID进行挂载（blkid）

4.LVM磁盘管理

4.1 什么是LVM？作用是什么？

LVM（Logical Volume Manager，逻辑卷管理）是 Linux 系统中一种灵活的磁盘管理技术。

  • 灵活性高：传统磁盘分区后大小固定，而 LVM 的逻辑卷可以动态扩展或缩小，无需重新分区或重启系统。例如当某个分区空间不足时，可以直接从其他物理卷扩容。
  • 便于管理：可以动态添加或移除物理卷，将多块硬盘统一管理。比如需要扩容时，只需添加新硬盘并加入卷组，再扩展逻辑卷即可。
  • 提高效率：PE（物理扩展块）的灵活大小设置可以优化硬盘利用率。PE 越小利用率越高，但同时也限制了卷组最大规模。
  • 空间利用率优化：避免了传统分区时预留过多空间或空间浪费的问题，可以根据实际需求动态调整各逻辑卷的大小。
  • 防止磁盘满：可以结合磁盘配额功能限制用户使用空间，防止某个用户占用过多磁盘资源导致系统故障。

简单来说，LVM 就像给硬盘加了一个”中间商”，让你可以更灵活地分配和管理存储空间，特别是在服务器环境中非常实用。

4.2 LVM三层结构介绍

PV（物理卷 - Physical Volume）

物理卷是 LVM 机制的基本存储设备，通常对应一个普通分区或整个硬盘。

特点：

  • 创建物理卷时，会在分区或硬盘头部创建保留区块，记录 LVM 属性
  • 存储空间分割成默认大小为 4MB 的基本单元（PE - Physical Extent）
  • PE存储单元是磁盘中一个固定大小的存储块 ，可选值：4、8、16、32、64 MB
  • PE 越小硬盘利用率高，但每个 VG 最多只有 65534 个块，所以 PE 大小决定了 VG 卷组大小

示例： /dev/sdb1、/dev/sdc1

VG（卷组 - Volume Group）

由一个或多个物理卷组成的整体，可以动态地添加或移除物理卷。

LV（逻辑卷 - Logical Volume）

逻辑卷建立在卷组之上，从卷组中”切出”一小块空间，作为创建文件系统的基础。

特点：

  • 与物理卷没有直接关系
  • 对于逻辑卷来说，每个卷组就是一个整体
  • 使用 mkfs 等工具在逻辑卷上创建文件系统

5.RAID

5.1 什么是RAID？作用是什么？

RAID（Redundant Array of Independent Disks，独立磁盘冗余阵列）是一种通过组合多个磁盘驱动器，以实现数据冗余存储和性能提升的技术。

简单来说，RAID 就是将多块硬盘组织在一起，通过某种方式工作，使得这个硬盘组合在容量、速度或安全性方面都优于单块硬盘。

RAID的作用：1+1>2

  • 提高读写性能：多块硬盘可以并行读写，大幅提升 I/O 性能，是单个硬盘性能的多倍。
  • 数据冗余保护：通过校验机制或备份，当某块硬盘故障时，数据不会丢失，提高了系统的可靠性和容错能力。
  • 增加存储容量：多块硬盘组合可以提供更大的总存储空间。
  • 提高系统可用性：在硬盘故障时，系统可以继续运行，无需中断服务，特别重要的是在生产环境中。

5.2 主要RAID级别介绍

RAID 级别 硬盘数量 磁盘利用率 是否有校验 保护能力 写性能
RAID 0 N N 无 无保护 单个硬盘的 N 倍
RAID 1 N（偶数） N/2(50%) 无 允许一个设备故障 需写两份存储设备，互为主备
RAID 5 N≥3 (N-1)/N 有 允许一个设备故障 需写计算校验
RAID 6 N≥4 (N-2)/N 有 允许两个设备故障 需双重写计算校验
RAID 10 N≥4（偶数） N/2(50%) 无 允许两个基组中各坏一个 N/2 块盘同时写入

RAID 0（N块盘组成，条带化——同时读写）

特点：

  • 高性能读写(I/O)最好，速度是单硬盘的 N 倍(有几块就几倍)

  • 数据分散存储在多块硬盘上，没有冗余，硬盘利用率最高（100%）

风险：

  • 无备份，任何一块硬盘故障，整个 RAID 数据全部丢失
  • 不适合存储重要数据

适用场景：临时数据、缓存、对容错无要求的场景

RAID 1（N/2偶数块组成，镜像——相互备份）

特点：

  • 数据完全备份，互为主备（允许坏任意一块）
  • 每次写操作需要写两份数据，写的性能是一块盘，读的性能是接近N块盘

缺点：

  • 磁盘利用率低，占用空间是N块盘的空间，成本高

适用场景：系统盘、数据库、对数据安全要求极高的场景

RAID 5（N>=3块组成，N-1块做条带化 + 留一个做校验盘[用户数据恢复]）

特点：

  • 读写性能，写的性能稍弱，因为有校验盘的原因，读的性能很高（N-1块盘的性能）
  • 有一块盘备份（允许损坏任意一块）

缺点：

  • 写性能需要计算校验盘，比 RAID 0 低
  • 整体更贵了一点，整体的磁盘利用率是N-1/N的性能

适用场景：大多数生产环境的首选方案

RAID 6（N>=4块组成，双校验）

特点：

  • 使用双重校验机制，把N-2块盘做条带，剩下两个做校验盘（用户数据恢复的）称之为奇偶校验
  • 有两块校验盘，允许两块硬盘同时故障
  • 读写性能，写的性能相较RAID 5 更弱一点，因为有2块校验盘的原因，读的性能很高（N-2块盘的性能）

缺点：

  • 计算校验时间长，写性能受影响
  • 整体比RAID还贵，整体的磁盘利用率是N-2/N的性能

适用场景：超大规模存储系统、对数据极度重视的场景

总结

首先，讲述了磁盘相关知识，磁盘管理从分区开始，主分区最多 4 个（分区 ID：83 Linux、8e LVM、fd RAID、82 Swap），扩展磁盘需要创建逻辑磁盘后才能进行正常使用。

其次，从文件系统入手，磁盘要通过格式化文件系统、挂载（/etc/fstab 实现永久挂载）、配置后才能进行使用。

然后讲述了，LVM相关知识，LVM 提供 PV→VG→LV 三层结构，支持动态扩容（vgextend→lvextend→xfs_growfs）等。

最后又介绍了RAID，RAID 主要用于数据保护和性能提升，生产环境通常选 RAID 5（3+ 硬盘、容错 1 个、利用率 (N-1)/N），RAID 1 成本高安全性最高，RAID 0 性能最快但无容错能力。
