Shell相关知识

一、Shell 基础概念

1.1 什么是 Shell？

Shell 是一种弱类型的小脚本语言，最简单的 Shell 脚本就是把多条命令组合在一起，一次性执行。

Shell 的核心作用：

  • 命令解释器：解释并执行用户输入的命令
  • 脚本执行器：批量执行预定义的命令序列

1.2 Shell 解释器类型

解释器 路径 说明
Bash /bin/bash CentOS 7 默认解释器
Sh /bin/sh CentOS 7 之前的默认解释器
Csh /bin/csh C 语言风格的 Shell

1.3 Shell 应用场景

  • ✅ 重复性操作自动化
  • ✅ 交互性任务处理
  • ✅ 批量事务处理
  • ✅ 服务运行状态监控（巡检）
  • ✅ 定时任务执行

case 语句用于处理多个条件分支，类似于其他编程语言中的 switch 语句。当需要判断一个变量是否匹配多个固定值时，case 语句比 if-elif-else 更简洁高效。

十三、Shell 函数

13.1 函数的概念与作用

函数是一系列操作的集合，是 Shell 脚本中重要的代码组织方式。

使用函数的优点：

  • 避免代码重复，提高开发效率
  • 将复杂的大脚本分割为若干小的功能模块，提高代码可读性和维护性
  • 灵活组合多个函数，实现复杂功能
  • 符合企业级 Shell 编写规范和标准

💡 小知识：看一个人编写的 Shell 脚本有没有规范，就看是否大量使用函数。全是代码没有函数的人，说明还是初级水平。

  • 外层循环控制排序轮数（数组长度 - 1）
  • 内层循环比较相邻元素，较大的元素后移
  • 每轮排序后，最大的未排序元素会被移到正确位置

十七、总结

Shell脚本学习是一个从简单到复杂、从零散到体系的过程。

基础阶段涵盖了Shell的基本概念、脚本规范、管道重定向和变量运用，这是编写任何脚本的基石。逻辑控制阶段通过条件判断和循环语句，让我们能够处理复杂的业务逻辑，解决实际问题。代码组织阶段引入函数和数组，使脚本更加模块化、可维护，符合企业级开发标准。

Shell脚本的强大之处在于其自动化能力——一次编写，重复执行；在于其快速迭代——几分钟完成的脚本可能为你节省数小时的手工操作；在于其灵活组合——通过管道符和命令组合，实现强大而优雅的解决方案。

无论是系统运维、日志分析、数据处理还是服务管理，Shell脚本都是最直接、最高效的工具。关键是持续实践，在具体项目中应用这些知识，逐步积累经验，编写出越来越高质量的脚本。

磁盘相关知识1 磁盘分区1.1 首先需要了解，什么是磁盘分区？为什么要进行磁盘分区？1、磁盘存储空间无法直接使用，需要先进行分区（划分存储空间）才能使用磁盘容量。分区的主要目的是将一块完整的磁盘逻辑地分割成多个独立的存储区域。2、主分区只能创建4个（/dev/sda1 /dev/sda2 /dev/sda3 /dev/sda4）磁盘分区创建好后，需要格式化就能使用其空间了。 主分区，主要存储 ： ① 系统文件 ② 程序文件 ③ 用户数据 3、剩余的空间可以自定义控制，但是得先化分扩展分区比如，一块磁盘一共有1000G，主分区用了400G，还剩余600，此时我想使用剩余的600G空间，需要先把600G的空间定义为扩展分区（扩展分区不能直接存储数据），需要把扩展分区中的存储空间，进一步创建为1个或多个逻辑空间之后，才能作为存储设备使用。 1.2 主分区（Primary Partition） 主要存储系统文件、程序文件、用户数据 Linux系统最多只能创建4个主分区 命名规则：/dev/sda1、/de...
