设置Git的换行符处理

问题背景： 不同操作系统处理文本文件换行的方式不一样：

  • Windows：用两个字符换行（回车+换行，简称 CRLF）
  • Linux/macOS：只用一个字符换行（换行符，简称 LF）

当你和使用不同操作系统的人一起协作开发项目时，这种差异可能会导致文件格式混乱，Git 会误以为文件被修改了。

解决方法： 因为你用的是 Ubuntu（Linux 系统），运行这个命令： 如果是windows—则core.autocrlf true

这个命令的作用：

  • 当你提交代码时：自动把 CRLF 转换成 LF（统一格式）
  • 当你拉取代码时：保持原样不变（因为你的系统本来就用 LF）

这样可以保证代码仓库里的文件都使用统一的换行符格式（LF），避免因为换行符不同而产生的问题。

三、处理文件和暂存区

暂存区

为什么需要暂存区？

暂存区就像你的手提箱——你可以向其中添加物品（更改），如果你改变主意，可以删除物品，当你对手提箱里的所有东西感到满意时，你就把它拉上拉链（提交）。

通过git add命令告诉Git跟踪文件后，Git会将其添加到暂存区，包含到下一次的提交中

忽略特定文件.gitignore

有时，我们可能不希望 Git 跟踪某些文件，例如临时文件或敏感信息。Git 允许使用名为**.gitignore**的特殊文件忽略这些文件。

将问价存入**.gitignore**中即可让Git忽略其中的文件

可以看见输出中并没有列出debug.log

.gitignore文件非常强大。它可以忽略特定文件、整个目录，或使用模式匹配来忽略与特定条件匹配的文件。这在实际项目中非常有用，在这些项目中，可能有不应成为 Git 存储库一部分的构建工件、缓存文件或特定于环境的配置文件。

查看更改

Git提供了git diff命令，让我们能在提交更改之前查看更改

取消暂存更改

如果我们在提交项目前改变主意，不想在下次提交中提交包含这些更改的文件，可以使用git restore命令来取消暂存文件

例如取消hello.py的暂存提交

可以看见使用restore命令后，hello.py已经转移到了未暂存提交的更改下

小结

暂存区域，它是一项强大的功能，它允许我们通过仔细选择要包含的更改来制作更有意义的提交。暂存区重要的原因：

  1. 选择性提交：暂存区域允许我们仅提交部分更改，从而帮助我们创建更小、更有针对性的提交。
  2. 忽略文件：通过**.gitignore**排除不需要版本控制的文件来帮助保持存储库干净。
  3. 审查更改：git diff允许您在提交之前仔细检查更改，帮助您及早发现错误。
  4. 灵活性：取消暂存更改的能力使您可以自由地改变主意，从而使 Git 使用起来不那么令人生畏。

四、Git分支操作

分支的作用：
隔离开发 - 多人在不同分支独立工作，互不影响

特性开发 - 为新功能创建专门分支，完成后合并

bug 修复 - 快速创建修复分支，解决问题后合并

版本管理 - 用不同分支代表不同版本（如 main、develop）

代码审查 - 通过 Pull Request 审查代码后再合并

并行开发 - 多个分支同时进行，提高效率

实验空间 - 创建临时分支尝试，不满意可删除

删除不需要的分支

如果尝试删除尚未完全合并的分支，Git 会发出警告。在这种情况下，如果您绝对确定要删除分支，则可以使用选项代替-D 。这将强制删除分支，而不管其合并状态如何。

清理旧分支可以让我们的仓库井井有条，尤其是在处理具有许多替代现实的大型项目时。这可以保持我们的工作空间干净并防止混乱。

小结

分支作用：隔离开发、特性开发、bug修复、版本管理、代码审查、并行开发

分支是Git中实现团队协作、版本管理和高效开发的关键工具，合理使用分支能显著提高开发效率和代码质量。

git revert 创建一个新提交来撤消指定提交的更改，不会删除原始提交。相比 git reset 的破坏性，git revert 更安全，适合已推送的提交。

小结

掌握的高级 Git 操作：

  1. 修改提交：修复小错误或添加小改动，无需创建新提交，保持历史干净
  2. 恢复提交：创建新提交撤销更改，非破坏性方法，适合协作环境
  3. 挑选提交：精确选择特定提交应用到其他分支，灵活控制合并
  4. 交互式变基：重新排序、合并、编辑提交，在共享前清理本地历史
  5. 根变基：重塑整个项目历史，提供完全控制但风险极高

这些工具能维护清晰、有序的提交历史，提升代码审查效率、便于调试和理解项目演进。关键是：本地操作保持灵活，共享分支谨慎为上。

小结

掌握的核心概念：

  1. 存储更改：用 git stash 临时保存未完成的工作
  2. 应用更改：用 apply 保留存储，用 pop 删除存储
  3. 从存储创建分支：用 stash branch 将隐藏的改动转为新分支
  4. 管理多个存储：用消息标识，需要时查看和应用特定存储
  5. 清理存储：用 drop 删除单个或 clear 删除全部

使用要点

  • Stash 用于快速上下文切换，无需提交未完成的工作
  • 始终为存储添加描述性消息便于管理
  • 不要积累过多存储，长期工作改用分支或提交
  • Stash 是临时方案，不适合长期存放工作

Linux Centos7配置本地yum仓库引言本地yum仓库是什么？yum是Linux中常用的包管理工具，它通过访问”软件仓库“来下载、安装、更新、卸载软件包。 yun本地仓库就是把软件包(.rpm)和仓库的元数据都存放在本地磁盘(或者局域网服务器)上，而不是从外网获取的仓库。 yum仓库是干什么的? 离线环境安装软件，例如不能联网的服务器（内网机房、保密环境）。 节省带宽和时间，局域网里多台机器都要装软件时，可以共用一个本地仓库，避免每台都从外网重复下载，速度更快。 稳定可控，外网源可能更新、下线或变更版本。本地仓库内容是固定的，适合企业环境中版本可控、可追溯的管理。 自定义仓库，可以自己选择要放哪些包，甚至自己编译 .rpm 包后放进去，形成企业内部定制仓库。 为什么要用本地yum仓库？简而言之：因为某些场景下，不能或不方便使用外部网络源。 使用场景 原因 服务器无外网访问权限 无法连接公网源，只能用本地仓库 局域网多台机器部署 提高效率，共享本地仓库 企业安全与版本控制 防止自动升级造成不兼容 离线环境（如实验室、军工内网） 没有互联网，只能靠U...
