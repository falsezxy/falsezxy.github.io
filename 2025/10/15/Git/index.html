查看git状态

  • “分支主”：将分支想象成时间线。“主”是项目的主要时间线。
  • “尚未提交”：提交就像视频游戏中的保存点。我们还没有做出任何保存点。

小结

通过上述的内容，需要了解：

  1. 如何使用git init
  2. 如何使用git status
  3. 如何使用git add
  4. 如何使用git commit
  5. 如何使用git log

同时需要了解，为什么要使用Git?

  1. 详细历史记录：使用 Git，您可以拥有项目的完整历史记录。您可以看到发生了什么变化、何时发生了变化以及为什么发生了变化。
  2. 实验：Git 允许您自由实验。您可以创建项目的不同版本，而不必担心丢失您的原始作品。
  3. 协作：在团队中工作时，Git 可以轻松查看谁进行了哪些更改以及为什么进行了更改。
  4. 备份：每个 Git 存储库都是项目的完整备份，包括其整个历史记录。

设置Git的换行符处理

问题背景： 不同操作系统处理文本文件换行的方式不一样：

  • Windows：用两个字符换行（回车+换行，简称 CRLF）
  • Linux/macOS：只用一个字符换行（换行符，简称 LF）

当你和使用不同操作系统的人一起协作开发项目时，这种差异可能会导致文件格式混乱，Git 会误以为文件被修改了。

解决方法： 因为你用的是 Ubuntu（Linux 系统），运行这个命令： 如果是windows—则core.autocrlf true

这个命令的作用：

  • 当你提交代码时：自动把 CRLF 转换成 LF（统一格式）
  • 当你拉取代码时：保持原样不变（因为你的系统本来就用 LF）

这样可以保证代码仓库里的文件都使用统一的换行符格式（LF），避免因为换行符不同而产生的问题。
