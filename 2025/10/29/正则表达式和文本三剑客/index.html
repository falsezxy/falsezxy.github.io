正则表达式和文本三剑客

一、正则表达式基础

1.1 正则表达式概念

正则表达式是一种强大的文本模式匹配工具，用来描述、匹配和查找符合某种规则的文本内容。

重要概念：

  • 正则表达式用于模式匹配和文本搜索
  • 不同工具支持的正则表达式语法有所差异
  • 基础正则表达式（BRE）和扩展正则表达式（ERE）

⚠️ 重要区别：正则表达式的元字符与 Shell 通配符不同

  • 通配符：用于文件名匹配（*、?、[]）
  • 元字符：用于文本内容匹配（在 grep、sed、awk 中使用）

1.2 基础正则表达式元字符（BRE）

元字符 含义 示例 匹配情况
. 匹配任意单个字符（除换行符） g.d god、gad、g3d
* 匹配前面的子表达式 0 次或多次 go*d gd、god、gooood
^ 匹配行的开始 ^root 行首为 root
$ 匹配行的结束 bash$ 行尾为 bash
[] 字符集，匹配其中任意一个字符 [abc] a、b 或 c
[^] 否定字符集，匹配不在其中的字符 [^abc] 除 a、b、c 外的字符
\ 转义符，使特殊字符失去特殊含义 \. 实际的点号
\{n\} 匹配前面的子表达式恰好 n 次 go\{2\}d good（恰好2个o）
\{n,\} 匹配前面的子表达式至少 n 次 go\{2,\}d good、gooood
\{n,m\} 匹配前面的子表达式 n 到 m 次 go\{2,4\}d good、gooood

1.3 扩展正则表达式元字符（ERE）

扩展正则表达式需要使用 grep -E、egrep 或 sed -r 启用，不需要转义 {}。

元字符 含义 示例 匹配情况
+ 匹配前面的子表达式至少 1 次 go+d god、good
? 匹配前面的子表达式 0 次或 1 次 go?d gd、god
` ` 或关系，匹配左边或右边 `cat
() 分组，将多个字符作为一个整体 (go)*d d、god、gogod
{n} 匹配前面的子表达式恰好 n 次 go{2}d good
{n,} 匹配前面的子表达式至少 n 次 go{2,}d good、gooood
{n,m} 匹配前面的子表达式 n 到 m 次 go{2,4}d good、gooood

1.4 字符类速查表

字符类 含义 示例
[a-z] 小写字母 [a-z]
[A-Z] 大写字母 [A-Z]
[0-9] 数字 [0-9]
[[:alpha:]] 字母（等同于 [a-zA-Z]） [[:alpha:]]
[[:digit:]] 数字（等同于 [0-9]） [[:digit:]]
[[:alnum:]] 字母和数字 [[:alnum:]]
[[:space:]] 空白字符（空格、制表符等） [[:space:]]
[[:upper:]] 大写字母 [[:upper:]]
[[:lower:]] 小写字母 [[:lower:]]

三、sed 流编辑器

3.1 sed 概念

sed 是流编辑器（Stream EDitor），用于对文本进行自动编辑和转换。与 vim 交互式编辑不同，sed 是非交互式的。

工作原理：

  1. 读取：从输入流中读取一行内容存储到模式空间（pattern space）
  2. 执行：执行 sed 命令处理模式空间中的内容
  3. 显示：输出修改后的内容到屏幕
  4. 重复：不断重复上述过程，直到文件末尾

重要特性：

  • 默认情况下，输入文件内容不会被改变，除非使用 -i 选项或重定向
  • sed 是行编辑器，一次处理一行内容

3.2 sed 常用选项

选项 含义 说明
-e 指定编辑命令 一般在执行多个命令时使用
-f 从脚本文件中读取命令 sed -f script.sed file.txt
-n 仅显示处理后的结果 禁止默认输出，常与 p 命令搭配
⭐-i 直接修改源文件 ⚠️ 谨慎使用，建议先备份
-r 使用扩展正则表达式 使用 {n,m} 时不需要转义

四、awk 数据处理工具

4.1 awk 概念

awk 是一种强大的文本处理编程语言，用于对结构化数据进行提取、转换和输出。

工作原理：awk 过滤的是文本的列（字段），是一种流数据处理工具。

特点：

  • 默认以空格或 Tab 键作为分隔符
  • 多个连续空格会自动压缩为一个
  • 自动按行读取并处理数据
  • 支持内置变量和函数

4.2 awk 基本语法

参数说明：

  • -F "分隔符"：指定输入的字段分隔符（默认为空格或 Tab）
  • '模式 {操作}'：必须用单引号括起来
  • 模式：指定匹配的条件（可选）
  • 操作：指定执行的动作（放在 {}内）

4.3 awk 执行流程

awk 有三个重要的执行阶段：

阶段 含义 用途
BEGIN{} 读取文件前执行 变量初始化、打印表头
{} 对每行执行 数据处理、匹配过滤
END{} 读取文件后执行 统计结果、打印总结

八、文本处理命令对比

处理对象对比

工具 处理对象 特点 主要用途
grep 行 过滤输出 查找包含特定内容的行
