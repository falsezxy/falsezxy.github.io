Ansible 核心概念

什么是 Ansible？

Ansible 是一个基于 Python 开发的开源自动化工具，用于进行配置管理、应用部署、任务编排等工作。它的核心理念是”无Agent”架构，通过 SSH 协议对远程主机进行管理。

Ansible 的核心特性

特性 说明
Agentless 无需在被管理主机上安装任何代理程序
SSH连接 基于 SSH 进行远程连接和操作
幂等性 多次执行相同操作，结果一致
模块化 工作基于模块，每个模块执行特定任务
批量操作 可同时管理成百上千台主机
易于扩展 支持自定义模块和插件

Ansible 工作流程

核心概念

  1. Inventory（主机清单）：定义要管理的主机和主机组
  2. Playbook（剧本）：定义执行的任务序列
  3. Module（模块）：执行具体操作的工具
  4. Task（任务）：调用模块的单个操作
  5. Handler（处理器）：在特定条件下触发的操作
  6. Role（角色）：组织任务、变量、文件的结构化方式

Ansible Playbook

什么是 Playbook？

Playbook 是 Ansible 的配置文件，用声明性语言定义一系列任务，是 Ansible 强大功能的核心体现。

Handlers 处理器

Handlers 是特殊的任务，只在被 notify 触发时执行。重要特性：

  • Handlers 在当前 Play 的所有任务完成后才执行
  • 多次 notify 同一个 Handler，Handler 仍只执行一次
  • Handler 名称必须唯一

面试题

基础概念类

Q1: Ansible 与其他配置管理工具（如 Puppet、Chef）相比有什么优势？

A:

  • Agentless：无需在被管理主机上安装代理程序，只需 SSH
  • 易于学习：使用 YAML 语法，学习曲线平缓
  • 快速部署：没有复杂的初始化过程
  • 幂等性：多次执行结果一致，安全可靠
  • 模块丰富：官方提供大量内置模块

Q2: 什么是 Ansible 的幂等性？举例说明。

A: 幂等性指多次操作对系统资源的影响是一致的。例如：

无论执行多少次，httpd 服务都是启动状态，第一次执行会启动服务（changed），后续执行检查到服务已启动则不做操作（ok）。而 systemctl restart httpd 是非幂等的，每次都会重启。

Q3: Ansible 如何实现无 Agent 的远程管理？

A: Ansible 通过以下方式实现无 Agent 管理：

  1. 利用 SSH 协议连接到远程主机
  2. 生成临时 Python 脚本和配置文件
  3. 通过 SSH 将文件传输到远程主机
  4. 在远程主机上执行 Python 脚本
  5. 返回执行结果
  6. 清理临时文件

Inventory 相关类

Q4: Ansible Inventory 有哪些定义主机的方式？

A:

  1. 直接 IP：192.168.1.10
  2. 主机名：web01 （需在 /etc/hosts 中定义）
  3. 指定端口：192.168.1.10:2222
  4. IP 范围：192.168.1.1[0:5] 表示 10-15
  5. 主机范围：db-[a:f].example.org 表示 a-f

Q5: Ansible 中变量的优先级是什么？

A: 从低到高依次为：

  1. defaults 目录（role 默认变量）
  2. group_vars 目录（组变量）
  3. host_vars 目录（主机变量）
  4. Playbook 中的 vars（Play 变量）
  5. 命令行 -e 参数（最高优先级）

Q7: Playbook 中 notify 和 handlers 的执行时机是什么？

A: Handlers 不是任务被触发后立即执行，而是在当前 Play 的所有任务完成后才统一执行。好处是：

  • 多次 notify 同一个 handler，handler 仍只执行一次
  • 避免不必要的重启（如配置文件多次改动，但只重启一次）

Q9: Playbook 中 with_items 和 loop 有什么区别？

A: 功能基本相同，但 loop 是 Ansible 2.5+ 推荐使用的现代语法。区别：

  • with_items 会对嵌套列表进行”拉平”处理
  • loop 保留列表结构
  • loop 更直观，推荐使用

Roles 相关类

Q11: 什么时候应该使用 Roles？

A:

  • 项目结构复杂，包含多个相关的任务
  • 需要代码复用，同一功能应用到多个项目
  • 团队协作，需要清晰的代码组织结构
  • 构建完整的服务栈（如 LAMP）

Q12: Roles 中 vars、defaults 和 group_vars 的区别是什么？

A:

位置 优先级 用途
roles/xxx/defaults/main.yml 最低 为 role 设置默认值，可被覆盖
roles/xxx/vars/main.yml 较高 role 内部使用的变量
/etc/ansible/group_vars/ 中等 为主机组定义变量
/etc/ansible/host_vars/ 较高 为单个主机定义变量
Playbook 中 vars 很高 Play 级别的变量
命令行 -e 最高 覆盖所有其他变量

Q14: 如何在生产环境中安全地使用 Ansible？

A:

  1. 权限管理

    • 避免使用 root 用户，使用专门的 ansible 用户
    • 配置 sudo 权限提升

  2. 密钥管理

    • 使用 SSH 密钥对而非明文密码
    • 使用 ansible-vault 加密敏感数据

  3. 版本控制

    • 将 Playbook 纳入 Git 版本控制
    • 建立 develop、staging、production 分支

  4. 执行限制

    • 使用 --check 模式进行干运行
    • 使用 --diff 显示变更
    • 使用 tags 灵活控制执行范围

总结

Ansible 作为现代化的基础设施即代码（IaC）工具，具有以下优势：

✅ 简单易用：YAML 语法，学习曲线平缓
✅ 无 Agent：只需 SSH，快速部署
✅ 幂等性：多次执行结果一致，安全可靠
✅ 强大灵活：丰富的模块和 Playbook 能力
✅ 生产就绪：广泛应用于企业级生产环境

通过掌握本笔记中的核心知识，结合实际项目经验，可以高效地构建自动化运维体系。
